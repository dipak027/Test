1. SELECT Basics

▼

📚 Theory

  • What it is: SELECT statement is used to retrieve data from one or more tables in a database
  • Purpose: It allows you to query specific columns, filter rows, and view data stored in tables
  • Key Components: SELECT (columns to retrieve), FROM (table name), WHERE (filtering conditions), ORDER BY (sorting)

Basic Syntax:

SELECT column1, column2 FROM table_name WHERE condition ORDER BY column1;

🎯 Example Problems (2-3)

Problem 1: Retrieve all employees

Retrieve all information about employees from the employees table.

Table: employees (employee_id, first_name, last_name, salary, department)
Expected Output: All columns and all rows from employees table

Problem 2: Specific columns with condition

Get first name and salary of employees who earn more than 50000.

Table: employees
Expected Output: first_name, salary columns where salary > 50000

Practice 1: Select all products

Write a query to retrieve all columns from the products table.

Table: products (product_id, product_name, category, price, stock_quantity)
Expected Output: All product details

Practice 2: Select specific columns

Retrieve only product_name and price from the products table.

Table: products
Expected Output: product_name, price columns only

Practice 3: Filter by price

Get products where price is greater than 100.

Table: products
Expected Output: Products with price > 100

Practice 4: Sort results

Retrieve all products ordered by price in descending order.

Table: products
Expected Output: All products sorted by price (highest first)

Practice 5: Multiple conditions

Find products in 'Electronics' category with price less than 500.

Table: products
Expected Output: Electronics products under $500
